# 项目路线图

本路线图描述交付顺序和阶段出口，不替代后续的文件级实施计划。交付原则是 CI 先行、生产 CD 后置：工程一旦可构建就进入持续验证，但在迁移、健康检查、备份和回滚可证明前，不给流水线生产部署权限。

状态：已完成，基线提交为 `faea211`。

产出：

- 根目录 README 和 AI 工作规则。
- 完整设计规格。
- 架构、领域、功能、UI、开发、部署、运维、安全和测试文档。
- 四条核心架构决策记录。

退出条件：

- 文档无相互矛盾的范围描述。
- 当前“尚未实施”的状态清晰。
- 其他 AI 能从文档找到任务对应模块和验收入口。

范围：

- 初始化 Git、Next.js Web、Worker 和共享包。
- 配置 TypeScript、Lint、格式化、测试框架和环境变量校验。
- 建立 PostgreSQL、Redis 和本地对象存储替身。
- 建立 CI 基线、开发 Compose 和镜像构建验证。
- Pull Request 从第一阶段开始执行 Lint、类型检查、测试、迁移和构建，不配置生产部署凭据。

已完成：workspace、Web/Worker 健康服务、环境校验、基线迁移、单元/集成测试基础、Playwright、生产形态镜像声明和两级 CI 门禁。

待完成：精确镜像 manifest 校验、隔离开发 Compose、本机容器 smoke 与依赖停止/恢复验证；CI 首次远端运行结果也需确认。2026-07-13 本机执行 `pnpm containers:smoke` 在镜像获取阶段 90 秒无进展后超时，未创建任何 smoke 容器，已清理 Compose 项目。

退出条件：

- 新环境可以按文档完成安装和启动。
- Lint、类型检查、测试和构建命令在 CI 通过。
- Web 与 Worker 镜像可在 CI 构建并完成容器级健康冒烟检查。
- 没有业务功能也能验证健康检查和数据库迁移。

## 阶段 2：身份、权限与内容核心

目标：打通管理员发布文章的最短闭环。

范围：

- 用户、凭据、会话、角色和状态。
- 管理员种子账号与安全登录。
- 文章、修订、分类、标签、媒体和发布状态。
- Markdown 编辑、预览、R2 上传和定时发布。
- 审计日志。

退出条件：

- 管理员可以创建、预览、发布、修改和归档文章。
- 普通用户无法访问管理操作。
- 迁移、权限和上传失败路径有自动化测试。

## 阶段 3：公开站点与原创视觉

目标：完成可公开阅读的品牌体验。

范围：

- “神乐的无月之境”欢迎场景和会话展示规则。
- 参考 Esunmo 的首页信息架构。
- 文章列表、详情、分类、标签、归档、搜索、RSS 和站点地图。
- 原创神乐静无月角色资产及响应式裁切。
- Live2D 看板娘客户端边界、静态回退、用户偏好和原创模型接入。
- 深色主题、减少动画、桌面和移动端适配。

退出条件：

- 首页和文章页通过桌面/移动端视觉检查。
- Core Web Vitals 不因欢迎动画或图片明显退化。
- 看板娘不阻塞 LCP，失败时正文和静态回退可用，移动端不自动下载模型。
- 图片授权与生成来源有记录。
- SEO、RSS 和站点地图可被自动化验证。

## 阶段 4：用户互动与后台治理

目标：开放受控的读者互动。

范围：

- 用户注册、登录、退出和用户中心。
- 点赞、收藏和评论。
- 评论全量审核。
- 用户禁言、封禁、解封和会话强制失效。
- 后台仪表盘、用户管理和评论队列。
- Turnstile 与 Redis 限流。

退出条件：

- 核心互动流程通过 Playwright。
- 重复点赞和收藏保持幂等。
- 封禁用户无法继续创建互动内容。
- 审核前评论不会出现在公开接口或页面。

## 阶段 5：每日热点

目标：上线可维护、可审核的热点聚合。

顺序：

1. 规范化契约、来源状态和候选审核。
2. Hacker News 与 GitHub Trending。
3. Bilibili、微博和百度适配器。
4. 当前热榜、北京时间每日归档和后台重试。

退出条件：

- 每个适配器均有固定响应样本测试。
- 任一来源失败不阻塞其他来源。
- 未审核候选不会公开。
- 每日归档可重复运行且不会产生重复数据。

## 阶段 6：生产交付

目标：在现有美国服务器上与现有服务安全共存，并在全部门禁具备后启用生产 CD。

范围：

- Web 与 Worker 生产镜像。
- GHCR、GitHub Actions 和受限部署用户。
- 生产 Compose、资源限制、健康检查和迁移任务。
- 宝塔 Nginx 反向代理与 HTTPS。
- 备份、恢复演练、日志轮转和回滚。
- SSH、Fail2ban 和防火墙加固方案。

退出条件：

- 合并 `main` 可自动部署。
- 健康检查失败会恢复上一个镜像版本。
- 数据库备份能在隔离环境恢复。
- 现有 `sub2api` 服务未受影响。
- root 密码已轮换，流水线不使用 root 密码。
- 只有本阶段退出条件满足后，`main` 工作流才获得生产 Environment 和部署 Key。

## 阶段依赖

- 阶段 2 依赖阶段 1。
- 阶段 3 和阶段 4 依赖阶段 2，可在内容契约稳定后并行。
- 阶段 5 依赖阶段 1 的 Worker 基础和阶段 2 的管理员审核能力。
- 阶段 6 在所有计划上线的 V1 功能通过验收后进行；阶段 1 的 CI 不等待阶段 6。

## 延后能力

以下能力不进入 V1：

- 邮箱验证和找回密码。
- GitHub、Google、QQ 等第三方登录。
- 签到、积分、等级、关注和排行榜。
- 自动 AI 摘要。
- 多作者投稿与审核。
- 独立搜索服务和微服务拆分。
