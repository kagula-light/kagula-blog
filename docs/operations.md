# 运维

## 运行目标

- 公开站点可用。
- 发布和评论审核数据不丢失。
- 热点来源失败不会影响文章阅读。
- 部署可以自动验证和回滚。
- 现有服务器服务不受博客运维影响。

## 健康检查

### 存活

`/api/health/live` 只判断应用进程是否响应，不查询所有依赖。

### 就绪

`/api/health/ready` 检查：

- PostgreSQL 可执行轻量查询。
- Redis 可用。
- 必需配置已加载。
- 当前迁移版本与应用兼容。

R2 或某个热点来源暂时失败不应让公开博客退出就绪。

## 日志

- Web 与 Worker 输出 JSON 结构化日志。
- 包含时间、级别、服务、版本、关联 ID 和事件名。
- HTTP 日志不记录密码、请求 Cookie 和完整 Authorization。
- Worker 错误只记录有限响应摘要。
- Docker 日志启用大小和文件数轮转。
- 审计日志保存在数据库，与运行日志分离。

## 备份

### 数据库

- 每晚执行一致性导出。
- 导出压缩并加密后上传独立 R2 备份桶。
- 上传成功并校验哈希后才删除本地临时文件。

保留策略：

- 最近 7 个每日备份。
- 最近 4 个每周备份。
- 最近 6 个每月备份。

### R2 资源

- 公开媒体通过对象版本或生命周期策略防止误删。
- 数据库备份桶与公开资源桶分离。
- 媒体数据库记录和对象清单定期比对。

### 配置

- Compose、Nginx 参考和脚本进入 Git。
- 生产 Secret 不进入 Git。
- 记录 Secret 创建与轮换日期，不记录 Secret 值。

## 恢复验证

至少每季度：

1. 在隔离环境创建空数据库。
2. 下载并解密随机备份。
3. 恢复数据。
4. 运行迁移兼容检查。
5. 验证用户、文章、评论和热点计数。
6. 记录耗时与结果。

未验证可恢复的备份不视为可靠备份。

## 日常巡检

每日自动检查：

- Web 就绪状态。
- 数据库备份结果。
- 五个热点来源最后成功时间。
- 磁盘使用率。
- 容器重启次数。

每周人工检查：

- 失败任务和死信。
- 数据库增长和慢查询。
- R2 孤立对象。
- 登录失败与限流异常。
- 依赖和基础镜像安全更新。

## 告警阈值

初始建议：

- 根磁盘使用超过 75% 预警，超过 85% 紧急。
- 连续两次数据库备份失败紧急。
- Web 就绪连续失败紧急。
- 单热点来源超过 6 小时未成功预警。
- 所有热点来源失败不影响博客，但需要紧急排查 Worker。
- 容器频繁重启或内存被系统终止紧急。

阈值在真实运行数据出现后调整。

## 事件处理

### Web 不可用

1. 检查 Nginx、容器状态和就绪接口。
2. 检查最近部署 SHA。
3. 如果与部署相关，回滚前一成功 SHA。
4. 如果数据库不可用，停止重复重启并保护数据。

### 数据库异常

1. 阻止写入流量或停止 Web/Worker 写操作。
2. 保存当前卷和日志。
3. 判断是连接、磁盘、迁移还是数据损坏。
4. 在隔离环境验证备份后恢复。
5. 未验证前不覆盖原卷。

### 热点来源失败

1. 查看来源错误与最后成功时间。
2. 暂停单个来源。
3. 使用固定响应样本重现解析问题。
4. 更新适配器并重新采集。
5. 不通过降低安全限制绕过来源封锁。

### R2 上传失败

- 文章可保存草稿，但不能引用未完成资源发布。
- 清理残留 `PENDING` 媒体。
- 恢复后重试上传，不在数据库伪造 `READY`。

## 维护窗口

普通兼容迁移走自动部署。需要长时间数据迁移、不可逆结构变化或服务器安全策略调整时，安排显式维护窗口并提前备份。
