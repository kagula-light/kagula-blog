# 测试策略

## 原则

- 测试与风险匹配。
- 领域规则优先使用快速单元测试。
- 数据库约束和基础设施使用真实依赖的集成测试。
- 核心业务闭环使用浏览器测试。
- 第三方热点使用固定样本，不让 CI 依赖外部网络。
- 任何“已完成”声明必须附实际命令与结果。

## 当前命令

~~~bash
pnpm lint
pnpm typecheck
pnpm test
pnpm test:integration
pnpm test:e2e
pnpm build
~~~

上述命令由根目录 `package.json` 提供。`pnpm test:e2e` 会启动 Web standalone 构建，因此必须先运行 `pnpm build`。

## 单元测试

当前代码通过 134 个 Web 单元测试、18 个 Worker 单元测试、20 个数据库单元测试、13 个配置单元测试、12 个 `@kagura/auth` 单元测试和 4 个契约单元测试，共 201 个 workspace 单元测试；数据库/R2 集成与 Playwright 由 CI 执行。

覆盖：

- 用户状态和权限策略。
- 密码与会话服务的非加密业务规则。
- 文章状态转换和定时发布判断。
- slug 和重定向规则。
- 评论审核状态。
- 点赞与收藏幂等。
- 热点规范化、指纹、去重和归档日期。
- 看板娘加载状态、关闭偏好和减少动画分支。
- 环境变量 schema。

## 集成测试

使用临时 PostgreSQL 和 Redis：

- 从空数据库执行全部迁移。
- 数据库唯一约束和外键。
- 事务发布文章与修订。
- 封禁用户与会话失效。
- 评论公开查询只返回已批准记录。
- 热点批次重复执行。
- Redis 限流与锁。
- R2 接口使用注入式 SDK 命令测试；真实对象存储集成使用 CI 或隔离环境的 S3 兼容服务。

当前身份集成测试覆盖迁移后 ADMIN/USER 会话解析、单会话撤销、BANNED 即时失效和 Redis 失败预算；只在 CI 或目标服务器隔离测试依赖中运行。

## 热点适配器

- 使用版本化固定响应样本。
- 正常、空、缺字段、结构变化、超大响应和超时。
- 适配器不得在测试中访问真实站点。
- 解析失败必须返回来源级错误。

## Playwright

当前已有 7 条 Chromium 测试：工程基础健康检查、未登录后台重定向、统一登录错误、ADMIN 登录与 HttpOnly/SameSite Cookie、退出后旧 Cookie 失效、USER 后台拒绝，以及管理员创建/预览/发布/归档文章。测试 global setup 只在隔离数据库 upsert `e2e_admin`/`e2e_user`、清理其文章并撤销旧会话，不进入应用 bundle。

核心流程：

1. 注册、登录、退出和会话过期。
2. 管理员创建草稿、上传封面、预览并发布。
3. 普通用户不能进入后台。
4. 用户点赞、收藏并在用户中心查看。
5. 用户提交评论，公开页暂不可见。
6. 管理员批准评论，公开页可见。
7. 管理员封禁用户，现有会话失效。
8. Worker 创建热点候选，管理员批准后前台出现。
9. 按日期查看热点归档。

## UI 与响应式

页面：

- 欢迎场景。
- Live2D 看板娘及静态回退。
- 首页。
- 文章详情。
- 热点当前与归档。
- 登录、注册和用户中心。
- 后台文章、评论、用户和热点页。

视口：

- 1440 × 900。
- 1280 × 720。
- 768 × 1024。
- 390 × 844。
- 360 × 800。

检查：

- 无水平页面溢出。
- 固定工具不遮挡正文。
- 长标题和长单词不溢出。
- 图片加载失败有稳定占位。
- 动态内容不改变关键布局尺寸。
- 减少动画模式停用非必要动画。
- 看板娘 Canvas 非空且尺寸稳定；加载失败时海报仍可见。
- 看板娘和气泡不遮挡正文、目录、评论及移动端主要控件。
- 路由离开后运行时、监听器和动画循环被清理。

看板娘额外验证三种资源条件：正常模型、故意失败的模型 URL、禁用脚本。桌面验证空闲后加载和关闭偏好；移动端及 `prefers-reduced-motion` 验证默认不发起模型请求。浏览器检查同时记录控制台错误、网络请求和 Canvas 像素结果。

## 无障碍

- 语义 landmark。
- 键盘完成导航、搜索、登录、评论和后台审核。
- 对话框和抽屉正确管理焦点。
- 图片 alt。
- 图标按钮可访问名称。
- 色彩对比。
- 错误提示与字段关联。

## 安全测试

- 未登录、USER、ADMIN、MUTED 和 BANNED 矩阵。
- 登录暴力限流。
- 注册 Turnstile 失败。
- CSRF。
- Markdown XSS。
- 评论危险内容。
- 伪装文件、超大文件和未授权删除。
- 热点非白名单 URL 与重定向。
- 开放重定向和敏感错误泄露。

## CI 门禁

Pull Request 必须通过：

- Lint。
- 类型检查。
- 单元测试。
- 迁移与集成测试。
- Web 与 Worker 构建。
- Chromium 冒烟测试。
- 生产形态 Web/Worker 镜像与就绪检查。

GitHub Actions 工作流 `CI` 包含两个顺序门禁：

- `quality`：格式、Lint、类型、单元/集成测试、构建和 Playwright。
- `container-smoke`：仅在 `quality` 成功后构建隔离镜像并验证 Web/Worker 就绪。

阶段 2A 的 PR #1 已通过上述两个门禁；运行 `29299762599` 覆盖 PostgreSQL/Redis 集成、6 条 Chromium 流程和 Linux 容器 smoke。该结果不代表目标服务器已经部署或完成隔离运行验收。

`pnpm containers:smoke` 会先等待 PostgreSQL、Redis、迁移、Web 和 Worker 全部成功，再单独运行验证容器。除 readiness 外，当前还验证未登录 `/admin` 返回带 `next=/admin` 的登录重定向；smoke 不创建管理员。任何启动或验证失败都会返回非零，并在清理 Compose 项目前写入 `container-smoke.log` 供 CI 上传。

CI 使用 PostgreSQL 17 与 Redis 7.4 服务容器和测试专用本地凭据，不访问生产环境。

影响核心流程或 UI 的变更还需要相关 Playwright 与截图检查。

## 验证报告

最终说明分为：

- 静态检查。
- 自动化测试。
- 构建。
- 本地真实运行。
- 生产或浏览器验证。

没有执行的项目必须明确说明，不用模糊的“已验证”覆盖缺口。
