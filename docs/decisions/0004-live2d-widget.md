# ADR-0004：使用按需加载的 Live2D 看板娘运行时

状态：已接受
日期：2026-07-10

## 背景

公开博客需要“神乐静无月”看板娘。目标体验有动画、对话气泡和陪伴感，但不能阻塞首屏、覆盖长文阅读或迫使移动设备自动下载大型模型。模型资产还存在独立于代码依赖的授权风险。

## 决策

- V1 首选 `hacxy/l2d-widget`，在脚手架阶段复核版本并由锁文件固定。
- 运行时封装在独立 Client Component 和适配器中，页面不直接调用第三方 API。
- 服务端始终先渲染静态海报和稳定占位。
- 桌面端在欢迎场景结束与浏览器空闲后按需加载；移动端和减少动画模式默认不自动加载。
- 用户可以休眠、关闭并保留偏好，组件卸载时必须销毁运行时。
- 模型及其全部衍生资源必须原创或授权清晰，并托管在自有 R2 资源域。

## 结果

优点：

- 看板娘故障与页面内容隔离。
- 初始渲染和 SEO 不依赖 Canvas。
- 生命周期接口支持休眠、切换和清理。
- 用户能控制资源和干扰程度。

代价：

- 需要额外维护客户端加载状态、CSP 和资源清单。
- Live2D 模型制作、拆分和授权仍是独立工作。
- 第三方包较新，必须在真实模型、浏览器和销毁场景下验证。

## 退出条件与替代方案

若 `l2d-widget` 无法通过 CSP、按需加载、模型兼容、无泄漏销毁或性能验收，则暂停接入并记录后续 ADR。备用方案是基于 `pixi-live2d-display` 构建更低层适配器；不回退到不受控的全局 CDN 脚本或授权不明模型。

## 许可说明

库许可证不等于模型授权。发布条件和资产记录以 [Live2D 看板娘](../live2d-mascot.md) 为准。
