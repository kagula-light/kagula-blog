# ADR-0002：自托管账号与两角色权限

状态：已接受
日期：2026-07-10

## 背景

普通用户需要登录后评论、点赞和收藏。用户暂不希望申请邮件服务，并要求登录不是首要开发阶段，但权限基础必须可靠。

## 决策

- 使用用户名和密码。
- 密码使用 Argon2id。
- 使用数据库会话。
- V1 角色只有 `ADMIN` 和 `USER`。
- 用户状态为活动、禁言和封禁。
- 权限通过集中服务端策略判断。
- 邮箱、外部账号和验证时间仅预留数据扩展能力。

## 结果

优点：

- 不依赖 SMTP 或外部身份平台。
- 可以立即支持评论治理和用户封禁。
- 权限边界简单、可测试。

代价：

- V1 没有自助找回密码。
- 用户名密码注册更依赖限流与 Turnstile。
- 未来增加邮箱验证需要迁移已有用户流程。

## 未选择

- 完整 RBAC：当前只有一个作者，不需要角色和权限配置界面。
- 社交登录：受平台、区域和配置影响，不进入 V1。
- 用户名密码无服务端状态：无法可靠强制失效封禁用户会话。
