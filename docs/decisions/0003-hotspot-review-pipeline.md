# ADR-0003：热点候选审核流水线

状态：已接受
日期：2026-07-10

## 背景

项目需要技术和国内综合热点。部分来源没有稳定开放 API，自动采集可能出现错误、失效、版权和内容质量问题。

## 决策

- 每个来源使用独立适配器。
- 自动采集只产生候选。
- 候选规范化和去重后进入审核队列。
- 只有管理员批准的条目公开。
- 公开页包含当前榜单和北京时间每日归档。
- 只保存标题、排名、来源、链接和采集时间，不复制全文。

## 结果

优点：

- 来源失败和内容风险可隔离。
- 管理员控制公开质量。
- 适配器可以单独替换或暂停。
- 每日归档形成稳定历史快照。

代价：

- 需要管理员持续审核。
- 热点公开存在人工延迟。
- 国内来源变化仍需要维护适配器。

## 未选择

- 全自动发布：无法可靠控制错误和内容风险。
- 全手工录入：失去聚合效率。
- 抓取并转载全文：超出产品目标并增加版权风险。
