# 神乐的无月之境 V1 设计规格

日期：2026-07-10
状态：用户已逐段确认，等待书面规格复核

## 1. 摘要

“神乐的无月之境”是神乐静无月的个人博客。内容以编程、AI 和开发经验为主，也包含个人随笔。

V1 提供：

- 具有原创二次元视觉的公开博客。
- 自有管理后台和 Markdown 内容发布。
- 用户名密码登录、用户管理和两角色权限。
- 点赞、收藏和全部先审后发的评论。
- 自动采集、人工审核的每日热点。
- 原创神乐静无月 Live2D 看板娘与静态回退。
- Cloudflare R2 媒体存储。
- GitHub Actions 到美国服务器的自动化部署。

项目从空目录开始，没有需要兼容的现有应用代码。

## 2. 已确认事实

### 品牌

- 站点名：神乐的无月之境。
- 作者名：神乐静无月。
- 作者虚拟形象与站点角色为同一角色。
- 角色为银白长发、蓝紫眼睛、深色斗篷的原创女性角色。
- 场景为无月星空、漂浮书页、微弱星光和代码符文。
- 看板娘使用原创 Live2D 模型，依赖许可证与模型资产授权分别管理。

### 技术与基础设施

- Next.js、React 和 TypeScript。
- PostgreSQL、Redis 和 Cloudflare R2。
- Docker 与 Docker Compose。
- GitHub 托管代码；阶段 1 建 CI，阶段 6 通过安全门禁后启用 GitHub Actions 生产 CD。
- 目标服务器在美国，Ubuntu 24.04、2 核、约 2 GB 内存。
- 服务器已有宝塔、Nginx 和必须保留的 `sub2api` 容器。
- 域名在实现阶段不固定，通过 `APP_URL` 配置。

### 用户与内容

- 普通用户可以评论、点赞和收藏，不能发布文章。
- V1 只有 `ADMIN` 和 `USER`。
- 使用用户名和密码，不做邮箱验证、找回密码或第三方登录。
- 评论全部需要管理员审核。
- 文章使用 Markdown。
- AI 摘要首版由管理员手动填写，后续预留自动生成接口。

### 热点

- 自动采集候选，管理员决定是否公开。
- 展示当前热榜和按日期归档。
- 首批来源为 GitHub Trending、Hacker News、Bilibili、微博热搜和百度热搜。
- 只保存标题、排名、来源、链接和采集时间。

## 3. 目标与非目标

### 目标

- 打通从后台撰写到公开阅读的完整闭环。
- 为读者提供稳定、舒适的长文阅读体验。
- 让管理员能够治理用户、评论和热点。
- 让代码可以自动测试、构建、部署和回滚。
- 让其他 AI 仅依赖仓库文档即可安全接手。

### V1 非目标

- 多作者投稿。
- 邮件系统和通知。
- 社交登录。
- 签到、积分、等级、关注和排行榜。
- 自动 AI 摘要。
- 私信、实时聊天和评论通知。
- 微服务或独立搜索集群。
- 复制参考站代码或受保护视觉资产。

## 4. 架构

采用模块化单体：

~~~mermaid
flowchart LR
    Browser["浏览器"] --> Nginx["宝塔 Nginx"]
    Nginx --> Web["Next.js Web"]
    Web --> PostgreSQL
    Web --> Redis
    Web --> R2["Cloudflare R2"]
    Worker --> PostgreSQL
    Worker --> Redis
    Worker --> Sources["五个热点来源"]
~~~

### Web

一个 Next.js App Router 应用承载：

- 公开博客。
- 登录注册和用户中心。
- 管理后台。
- 面向浏览器的 API 和 Server Actions。

React 默认使用 Server Components。只有欢迎场景、Live2D 看板娘、轮播、编辑器、互动按钮和移动目录等浏览器交互使用 Client Components。

### Worker

独立进程处理：

- 热点采集。
- 规范化和去重。
- 定时发布。
- 北京时间每日归档。
- 过期任务与临时数据清理。

Worker 与 Web 共用稳定领域契约和数据库，不导入 React。

### 数据与文件

- PostgreSQL 保存所有核心业务事实。
- Redis 只保存限流、缓存、任务锁和可重建队列状态。
- R2 保存公开媒体。
- 独立 R2 备份桶保存加密数据库备份。

完整模块和依赖方向见 `docs/architecture.md`。

## 6. 公开站点

### 欢迎场景

- 每个浏览会话只显示一次。
- 显示北京时间、站点名、作者和短句。
- 使用原创角色和无月星空场景。
- 提供跳过。
- 支持 `prefers-reduced-motion`。
- 移动端使用独立竖版裁切并减少粒子。

### 首页

信息架构参考 Esunmo：

- 站点导航。
- 搜索和热门关键词。
- 推荐文章轮播。
- 最新发布、最近更新、最多浏览、最多点赞和最多评论。
- 图文文章流。
- 今日热点入口和预览。
- 页脚与站点统计。

不加入参考站的抽奖、签到、积分和排行榜。

### Live2D 看板娘

- 使用 `l2d-widget` 作为首选客户端运行时，封装在独立 Client Component 中。
- 服务端先输出静态海报；桌面端在欢迎场景结束和浏览器空闲后加载模型。
- 移动端与 `prefers-reduced-motion` 默认不自动下载模型，用户可以显式启动。
- 提供休眠、关闭和重新唤醒；关闭偏好保存在本地。
- 模型加载失败不影响正文、导航和互动，看板娘不得覆盖关键内容或控件。
- 模型、纹理、动作和表情必须为原创或具有明确商用授权，并托管在自有 R2 资源域。

详细方案见 `docs/live2d-mascot.md` 和 ADR-0004。

### 文章

- 顶部封面和面包屑。
- 作者、时间、字数、阅读时间、分类和互动数据。
- 手动 AI 摘要。
- Markdown 正文、代码块、表格、图片、标题锚点。
- 桌面端正文 + 右侧作者卡和吸附目录。
- 移动端单列，目录进入抽屉。
- 标签、许可说明、上一篇/下一篇、相关文章和评论。
- 浮动工具不遮挡正文。

### 发现

- 搜索。
- 分类与标签。
- 年月归档。
- RSS。
- Sitemap。
- Canonical、Open Graph 和结构化数据。

## 7. 认证、用户与互动

### 认证

- 用户名密码注册和登录。
- Argon2id 密码哈希。
- 数据库会话。
- 安全 Cookie。
- Redis 限流与 Cloudflare Turnstile。
- 管理员可以重置密码和强制退出。
- V1 没有自助找回密码。

### 权限

- `ADMIN`：内容、用户、评论、热点和设置管理。
- `USER`：阅读、点赞、收藏和提交评论。
- `MUTED` 用户不能评论。
- `BANNED` 用户不能建立会话，现有会话失效。
- 服务端权限策略是唯一最终判定。

### 互动

- 点赞和收藏按用户与文章唯一，重复请求幂等。
- 评论只能由登录用户提交。
- 所有评论初始为 `PENDING`。
- 只有 `APPROVED` 评论公开。
- 管理员可以拒绝或软删除评论。

## 8. 内容后台

### 仪表盘

- 文章状态统计。
- 待审核评论和热点。
- 用户状态。
- 热点来源健康。
- 最近审计事件。

### 文章与媒体

- Markdown 编辑和实时预览。
- 草稿自动保存和显式修订。
- 分类、标签、封面、摘要和 SEO。
- 立即发布、定时发布和归档。
- R2 上传、格式校验和响应式变体。

### 评论与用户

- 评论状态筛选和批量审核。
- 用户搜索、禁言、封禁、解封和会话强制失效。
- 管理员密码重置。

### 热点与设置

- 候选筛选、批准、拒绝、手动补录和排序。
- 来源启停、重试和健康信息。
- 查看每日归档。
- 品牌、导航和默认资源设置。
- 重要操作审计。

后台是紧凑、工作导向的管理界面，不使用公开站点的大图和重动画。

## 9. 热点流水线

1. 每 30 分钟创建来源级任务。
2. 每个适配器在白名单、超时和响应大小限制内读取来源。
3. 统一为来源、外部 ID、标题、链接、排名和采集时间。
4. 按外部 ID、规范化链接和标题指纹去重。
5. 保存为 `PENDING` 候选。
6. 管理员审核。
7. `APPROVED` 候选进入当前热榜。
8. 按 `Asia/Shanghai` 生成不可变每日快照。

单来源失败不影响其他来源或 Web。不得绕过验证码、登录墙或来源明确阻断。

## 10. 数据模型

核心实体：

- User、Credential、Session。
- Post、PostRevision、Category、Tag、MediaAsset。
- Comment、PostLike、Favorite。
- HotspotSource、HotspotCandidate、DailyHotspotArchive。
- SiteSetting、AuditLog。

关键状态：

- 用户：`ACTIVE`、`MUTED`、`BANNED`。
- 文章：`DRAFT`、`SCHEDULED`、`PUBLISHED`、`ARCHIVED`。
- 评论：`PENDING`、`APPROVED`、`REJECTED`、`DELETED`。
- 热点：`PENDING`、`APPROVED`、`REJECTED`、`EXPIRED`。

数据库保存 UTC，界面和热点归档使用北京时间。

完整约束见 `docs/domain-model.md`。

## 11. UI 与动效

### 色彩

- 中性深灰背景与表面。
- 绿色符文光作为主要操作与当前状态锚点。
- 蓝紫星光负责空间层次和品牌氛围。
- 品红只用于收藏等少量情感反馈。
- 成功、警告和危险使用不同色相。

### 动效

- transform 和 opacity 优先。
- 页面转场短且一致。
- 环境粒子 Canvas 仅在欢迎场景；看板娘 Canvas 独立按需加载。
- 移动端自动降级。
- 减少动画时停用视差、粒子和长转场。
- 动态内容使用稳定尺寸，避免布局偏移。

### 响应式

必须验证 1440×900、1280×720、768×1024、390×844 和 360×800。

文章移动端不得横向溢出，表格和代码块只能在自身容器滚动。

## 12. 安全

- 输入使用 schema 校验。
- Markdown 输出白名单清理。
- 上传验证文件头、MIME、尺寸和权限。
- 用户 SVG 默认拒绝。
- 会话、权限和用户状态在服务端校验。
- 热点采集只访问来源白名单，阻止 SSRF。
- Secret 分别保存在本地忽略文件、GitHub Environment 和生产受限环境文件。
- 管理操作写入审计日志。
- 日志不包含密码、Token、完整 Cookie 或第三方完整响应。

生产前：

- 轮换已暴露的 root 密码。
- 创建部署用户和 SSH Key。
- 验证后关闭 root 密码登录。
- 启用 Fail2ban。
- 在用户授权下收敛面板、FTP 和数据库防火墙规则。
- 不影响现有 `sub2api`。

## 13. CI/CD

### Pull Request

- Lint。
- TypeScript。
- 单元测试。
- PostgreSQL/Redis 集成测试与迁移。
- Web 和 Worker 构建。
- 按变更范围运行 Playwright。
- 构建 Web 与 Worker 镜像并做容器健康冒烟检查。

### main 生产 CD

阶段 1 先建立 CI 和镜像验证，但不配置生产权限。只有生产 Compose、迁移、备份、健康检查、回滚和受限部署用户在阶段 6 验证完成后，才启用以下自动部署：

- 构建 Web 和 Worker 镜像。
- 使用提交 SHA 标记并推送 GHCR。
- 通过受限部署用户连接服务器。
- 拉取镜像。
- 执行一次性迁移。
- 启动新版本。
- 检查 `live` 和 `ready`。
- 失败恢复前一成功 SHA。

服务器不执行源码构建。

## 14. 备份与运维

- 每晚加密 PostgreSQL 备份到独立 R2 桶。
- 保留 7 日、4 周、6 月备份。
- 每季度在隔离环境恢复验证。
- Docker 日志轮转。
- 监控健康、备份、磁盘、容器重启和来源最后成功时间。
- Web 失败可回滚镜像；数据库问题按备份恢复流程处理。

## 15. 测试与验收

### 自动化

- 领域状态、权限、幂等和配置单元测试。
- 迁移、约束、会话、Redis 和 R2 集成测试。
- 五个热点适配器固定响应样本。
- Playwright 覆盖注册登录、发布文章、评论审核、点赞收藏、封禁和热点发布。
- Playwright 覆盖看板娘正常加载、静态回退、关闭偏好、移动端和减少动画模式。

### UI

- 桌面、平板和手机。
- 正常动画和减少动画。
- 键盘导航和焦点。
- 长标题、长代码、表格和图片失败。
- 固定工具不遮挡。

### 完成报告

实施任务必须区分：

- 静态检查。
- 自动化测试。
- 构建。
- 本地运行。
- 浏览器或生产验证。

## 16. 实施顺序

1. 仓库、workspace、配置、测试、CI 和镜像构建验证基础。
2. 数据库、认证、权限和管理员引导。
3. 文章、修订、媒体和后台发布闭环。
4. 公开首页、文章、发现、原创视觉和 Live2D 看板娘。
5. 用户互动、评论审核和用户治理。
6. 热点契约、五个适配器、审核和归档。
7. 生产 Compose、部署权限、备份、回滚和加固，验收后启用生产 CD。

详细阶段和退出条件见 `docs/roadmap.md`。文件级原子实施计划在用户复核本规格后生成。

## 17. 文档交付

仓库必须长期维护：

- 根 README。
- 项目级 AGENTS。
- 文档索引与 AI 上手指南。
- 架构、领域、功能、UI、配置、开发、部署、运维、安全、热点和测试。

## 18. 验收定义

V1 可以交付上线的条件：

- 管理员可以安全发布 Markdown 文章和媒体。
- 公开站点符合已确认视觉与响应式规则。
- 原创看板娘按需加载、可关闭、可回退且不影响阅读性能。
- 用户可以注册登录、点赞、收藏和提交待审评论。
- 管理员可以治理评论和用户。
- 五个热点来源独立运行，审核后公开并生成每日归档。
- CI 自动测试、构建、部署、健康检查和回滚。
- 备份已实际恢复验证。
- 生产部署不影响现有服务器服务。
- 文档足以让新的开发者或 AI 不依赖对话历史继续工作。
