# UI 设计规范

## 设计方向

公开站点参考 [Esunmo](https://esunmo.com/) 的信息架构与文章布局：

- 全屏欢迎场景。
- 深色导航和内容表面。
- 搜索、推荐轮播和多维文章流。
- 文章详情的封面、正文与右侧目录。
- 移动端单列阅读。

参考只用于理解布局与交互。项目不复制其代码、Logo、文案、插画或受保护资产。

## 品牌

- 站点名：神乐的无月之境。
- 作者名：神乐静无月。
- 内容定位：技术与 AI 为主，个人随笔为辅。
- 气质：安静、神秘、克制，不使用营销式口号。

## 原创角色

神乐静无月同时是作者虚拟形象和站点角色：

- 银白长发。
- 蓝紫色眼睛。
- 深色斗篷。
- 星光与月相魔法元素。
- 背景为无月星空、漂浮书页、微弱星光和代码符文。

资产组至少包含：

- 桌面欢迎场景横图。
- 移动端欢迎场景竖图。
- 方形头像。
- 默认文章封面。
- 空状态小图。
- 社交分享默认图。

生成时不得指定模仿现有艺术家或动漫角色。每个资产记录生成工具、日期、原始提示和后期修改。

## 色彩

计划色板：

| 用途 | 色值 |
| --- | --- |
| 页面背景 | `#0c0c0e` |
| 主要表面 | `#171719` |
| 抬升表面 | `#232328` |
| 主要文字 | `#f2f2f4` |
| 次要文字 | `#a9a8af` |
| 符文苔光，主操作 | `oklch(0.74 0.16 150)` |
| 无月星紫，空间层次 | `oklch(0.70 0.14 275)` |
| 静月品红，情感反馈 | `oklch(0.68 0.17 340)` |
| 警告 | `oklch(0.78 0.15 85)` |
| 危险 | `oklch(0.65 0.20 25)` |

绿色负责行动，蓝紫负责空间，品红负责情感反馈，三者不得混成通用渐变。正文表面保持中性深灰。实现阶段需要在真实组件上校验对比度，再固化最终 token。

## 字体

- 中文正文优先系统无衬线字体，确保加载快和长文可读。
- 品牌与文章大标题可使用有授权的中文衬线字体子集。
- 代码使用等宽字体。
- 不按视口宽度缩放字号。
- 紧凑后台不使用大尺寸展示字体。

## 欢迎场景

- 高度使用稳定的 `100dvh`。
- 每个浏览会话显示一次，使用会话存储记录。
- 显示北京时间日期和时间。
- 主视觉第一屏直接出现，不放在卡片中。
- 提供明确的跳过按钮。
- 进入主站使用 opacity/transform 过渡，不触发布局跳动。
- 移动端使用独立竖版裁切和更少粒子。
- `prefers-reduced-motion` 下不播放粒子、视差和长转场。

## Live2D 看板娘

- 默认位于桌面端右下安全区，尺寸使用稳定约束，不能因气泡或动作改变页面布局。
- 看板娘进入前显示同角色静态海报，避免空白 Canvas 和突然跳入。
- 交互按钮使用清晰图标和 Tooltip，至少提供休眠、关闭和重新唤醒。
- 对话气泡简短且可消失，不覆盖文章标题、正文、目录和评论控件。
- 移动端默认不加载模型，仅保留不遮挡主要操作的启动入口；用户明确启动后仍受安全区和尺寸上限约束。
- 减少动画模式只显示静态形象，不自动播放待机、口型或粒子动画。
- 看板娘只建立陪伴感和反馈状态，不承担导航或关键业务操作的唯一入口。

详细加载和生命周期规则见 [Live2D 看板娘](live2d-mascot.md)。

## 首页结构

1. 欢迎场景。
2. 主导航：Logo、首页、文章、今日热榜、归档、关于、搜索、登录/头像。
3. 搜索区：搜索框与热门关键词。
4. 推荐轮播：固定高度，图片和标题变化不改变布局。
5. 文章筛选：最新、最近更新、最多浏览、最多点赞、最多评论。
6. 图文文章流。
7. 今日热榜预览。
8. 页脚与站点统计。

首页不复制参考站的抽奖、签到和积分入口。

## 文章详情

### 桌面

- 内容容器最大宽度约 1200–1280 px。
- 主栏约 760–820 px，侧栏约 280–320 px。
- 顶部先展示文章封面，再展示文章头部与正文。
- 右侧显示作者卡、吸附目录和热门文章。
- 目录只突出当前阅读章节。
- 浮动工具只保留目录、主题和返回顶部，避免遮挡。

### 移动端

- 正文单列，左右安全边距至少 16 px。
- 封面保持稳定宽高比。
- 侧栏内容移到正文后或按需抽屉。
- 目录按钮打开全屏或底部 Sheet。
- 表格允许局部横向滚动，页面本身不能横向溢出。
- 代码块有复制按钮和独立滚动。

### 正文

- 正文行宽控制在易读范围。
- 标题锚点可复制。
- 图片提供说明和延迟加载。
- 外部链接标识来源。
- AI 摘要首版由管理员手动填写，不显示伪生成状态。
- 文末包含标签、许可说明、上一篇/下一篇、相关文章和评论。

## 后台

- 后台不沿用公开站点的大图欢迎场景。
- 使用紧凑侧边导航、表格、筛选、抽屉和对话框。
- 动效仅用于状态变化和反馈。
- 重要操作使用确认对话框。
- 审核页面优先批量处理效率。

## 动效系统

- 优先使用 CSS transform 和 opacity。
- 页面转场持续时间保持短而一致。
- 环境粒子 Canvas 仅在欢迎场景加载；看板娘 Canvas 独立按需加载。
- 粒子密度按设备能力降低。
- 滚动入场只播放一次，不让正文段落反复闪动。
- 轮播和动态内容使用固定尺寸，避免累计布局偏移。
- 动画必须可中断，路由离开时清理监听器和帧循环。

## 可访问性

- 颜色对比满足 WCAG AA。
- 所有按钮、菜单、轮播和抽屉可键盘操作。
- 焦点样式清晰。
- 图标按钮提供可访问名称和 Tooltip。
- 装饰图片使用空 alt，内容图片使用描述性 alt。
- 不用颜色作为唯一状态提示。
- 欢迎动画跳过后焦点进入主内容。

## 验收视口

至少验证：

- 1440 × 900 桌面。
- 1280 × 720 小型桌面。
- 768 × 1024 平板。
- 390 × 844 手机。
- 360 × 800 小型手机。

每个视口检查欢迎场景、首页、文章详情、登录注册和后台关键页面。
