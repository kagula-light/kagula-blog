# 安全设计

## 保护目标

- 用户密码和会话。
- 未发布文章与后台能力。
- 评论和热点审核边界。
- R2 资源与数据库备份。
- CI/CD 凭据和生产服务器。
- 现有非博客服务。

## 主要威胁

- 暴力登录和自动化注册。
- 普通用户越权访问后台。
- Markdown 或评论 XSS。
- 恶意图片和超大文件上传。
- 热点采集引发 SSRF、无限响应或解析器崩溃。
- Live2D 模型包引入未授权资产、远程脚本或不受控资源路径。
- Secret 泄露。
- 数据库迁移或部署破坏现有服务。
- root 密码和公网管理端口暴露。

## 密码与会话

- 密码使用 Argon2id；当前参数为内存 19456 KiB、时间成本 2、并行度 1、输出 32 字节。
- 登录错误使用统一提示，不泄露用户名是否存在。
- 会话 Token 只在浏览器出现原值，数据库保存摘要。
- Cookie 使用 HttpOnly、SameSite=Lax、根路径和显式过期时间；`APP_URL` 为 HTTPS 时启用 Secure。
- 密码重置、封禁和管理员强制退出会撤销相关会话。
- V1 不提供没有邮箱验证的自助找回密码。

初始限流建议：

| 动作 | 限制维度 | 默认 |
| --- | --- | --- |
| 登录 | IP + 用户名 | 10 分钟 5 次失败 |
| 注册 | IP | 1 小时 3 次 |
| 评论 | 用户 + IP | 10 分钟 5 条 |
| 点赞/收藏 | 用户 | 1 分钟 60 次 |
| 搜索 | IP | 1 分钟 60 次 |

阈值可配置，但不能由浏览器控制。

当前登录限流以客户端地址和规范化用户名的 SHA-256 摘要为 Redis key，10 分钟最多 5 次失败；Redis key 和日志不包含原始用户名或地址。未知用户名仍执行模块缓存的 dummy Argon2id 校验，避免跳过密码工作。

当前注册入口先执行字段校验，再以客户端地址 SHA-256 摘要执行 Redis 一小时 3 次预算，随后调用 Cloudflare Turnstile。只有挑战通过后才计算 Argon2id 哈希，并在同一 PostgreSQL 事务中创建 `USER`、凭据和摘要会话；规范化用户名冲突返回统一页面消息。注册后跳转只接受与 `APP_URL` 同源的相对路径，失败响应会重置 Turnstile token。

## 授权

- 每个服务端用例显式声明所需权限。
- 页面重定向和隐藏按钮只是体验，不是安全边界。
- 后台 API 对普通用户返回拒绝。
- 用户状态在敏感操作时重新读取或通过短期缓存验证。
- 审计发布、审核、角色、用户状态和站点设置变化。

当前 `/admin` 由 Server Component 布局调用服务端会话解析与 `canAccessAdmin`；页面重定向不是未来管理 API 的替代品，新增管理用例仍必须各自执行服务端权限策略。

## 输入与输出

- 所有 API 使用 schema 校验。
- Markdown 渲染后使用允许列表清理。
- 评论仅支持纯文本或受限 Markdown。
- 外部链接增加安全的 `rel` 属性。
- 富媒体嵌入只允许明确支持的平台。
- 错误页面不返回堆栈、SQL 或第三方响应正文。

## 上传

- 校验文件头、MIME、扩展名和尺寸。
- 拒绝 SVG 或对 SVG 做严格清理；默认不接受用户 SVG。
- 限制单文件大小、用户频率和每日总量。
- 对象键由服务端生成，不使用原文件路径。
- 上传完成前为 `PENDING`，验证后才为 `READY`。
- 公开资源凭据不能读取备份桶。

## 热点采集

- 每个适配器写死允许协议和域名。
- 禁止跟随到内网、回环、链路本地和私有地址。
- 限制重定向次数、响应大小和总超时。
- 不执行来源页面中的脚本指令。
- 不保存第三方全文。
- 使用低频请求、清晰 User-Agent，并尊重来源条款和阻断。

## 浏览器安全

计划响应头：

- Content-Security-Policy。
- Strict-Transport-Security。
- X-Content-Type-Options。
- Referrer-Policy。
- Permissions-Policy。
- frame-ancestors 限制。

CSP 需要为 R2 图片、必要脚本和字体建立最小允许列表。不得为了方便使用宽泛的任意脚本策略。

## Live2D 与第三方前端运行时

- `l2d-widget` 版本由锁文件固定并进入依赖审计；生产不从公共 CDN 动态执行脚本。
- 模型、纹理、动作和表情托管在自有 R2 资源域，只允许配置受控资源根下的相对路径。
- 模型包发布前校验清单、文件类型、单文件大小、总大小和路径穿越。
- CSP 只允许实际需要的脚本、Worker、WASM 与资源域；若运行时需要额外策略，必须通过浏览器验证并记录原因，不能直接放宽为通配符。
- 依赖的 MIT 许可证不授予任何模型权利。模型、纹理、动作、表情、声音和角色设计必须分别保存授权或原创来源记录。
- 不加载来源不明的模型仓库，不把远程返回内容作为看板娘指令执行。

## Secret 管理

- 本地 Secret 进入忽略的环境文件。
- CI Secret 进入 GitHub Environment。
- 生产 Secret 进入服务器受限环境文件。
- 镜像和构建日志不得包含 Secret。
- 不把 Secret 写入 SiteSetting、数据库迁移或测试样本。
- 部署 Key、R2 Key、会话 Secret 和备份 Key 分开轮换。

## 服务器加固

已确认的风险：

- root 密码登录曾启用。
- SSH 公钥登录曾未启用。
- Fail2ban 曾未运行。
- 宝塔、FTP 和部分端口规则对公网开放。

加固动作需要用户对生产服务器明确授权，计划顺序：

1. 轮换已暴露的 root 密码。
2. 创建受限部署用户。
3. 配置并验证 SSH Key。
4. 保留独立救援会话后关闭 root 密码登录。
5. 启用 Fail2ban。
6. 依据实际用途收敛 FTP、面板和数据库防火墙规则。
7. 复核宝塔面板访问控制。

任何加固都不得让现有服务失联。

## 安全验证

- 未登录和 USER 无法访问后台用例。
- 封禁立即影响现有会话。
- CSRF、XSS、上传伪装和开放重定向有测试。
- 热点适配器拒绝非白名单 URL。
- 日志扫描不包含常见 Secret 格式。
- 依赖和容器镜像在 CI 扫描。
