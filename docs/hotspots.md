# 每日热点

## 目标

聚合技术与国内综合热点，向管理员提供候选，由管理员决定公开内容。热点功能不是全文转载器。

## 首批来源

| 来源 | 内容 | 接入策略 |
| --- | --- | --- |
| GitHub Trending | 热门仓库 | 解析公开 Trending 页面或稳定替代数据 |
| Hacker News | 技术新闻 | 使用官方 Firebase API |
| Bilibili | 热门视频 | 使用公开排行数据，适配器可替换 |
| 微博热搜 | 国内综合 | 解析公开榜单，随时可暂停 |
| 百度热搜 | 国内综合 | 解析公开榜单，随时可暂停 |

GitHub Trending、Bilibili、微博和百度的公开页面或接口可能变化。代码不能假设它们具有永久稳定契约。

## 统一契约

适配器输出：

~~~text
sourceCode
externalId
title
url
rank
score?          可选来源热度
category?       可选来源分类
capturedAt
rawFingerprint
~~~

适配器不决定是否公开，也不直接写每日归档。

## 调度

- 默认每 30 分钟启动一次批次。
- 每个来源一个独立任务。
- 同一来源同一时间最多一个任务。
- 超时、重试和响应大小按来源配置。
- 失败使用指数退避并进入来源健康记录。
- 管理员手动重试仍遵守并发和安全限制。

## 规范化与去重

1. 清理标题首尾空白和不可见字符。
2. 规范化 URL，移除已知追踪参数。
3. 优先使用来源 externalId。
4. 没有 externalId 时使用来源 + 规范化 URL。
5. 再使用来源 + 标题指纹防止短期重复。
6. 已拒绝候选在保留窗口内不重复创建。

不同来源报道同一事件默认保留为不同候选，因为来源和排名具有独立意义。管理员可以选择只批准其中一个。

## 审核

状态：

- `PENDING`：等待审核。
- `APPROVED`：可进入当前热榜。
- `REJECTED`：不公开并保留拒绝记录。
- `EXPIRED`：长时间未审核且已失去时效。

管理员可以：

- 批准或拒绝。
- 修改公开展示标题，但保留原始标题。
- 调整当前榜单顺序。
- 手动补录来源和链接。
- 暂停或恢复来源。

## 当前热榜与归档

- 当前热榜只查询已批准且在有效窗口内的候选。
- 页面按来源和分类筛选。
- 每日归档日界线为 `Asia/Shanghai`。
- 归档保存展示快照，不引用会变化的候选标题。
- 同一日期归档操作幂等。
- 历史归档不因来源后续下线而消失。

## 失败隔离

- 单来源失败不终止批次。
- 所有来源失败也不影响文章页面和登录。
- R2 不参与热点核心数据保存。
- Redis 丢失后可以从 PostgreSQL 重建任务。
- 解析器异常只记录有限上下文。

## 合规与礼貌

- 只保存必要元数据，不复制全文、评论或媒体。
- 公开页面清楚标识来源并链接原文。
- 请求频率保持低且固定。
- 使用明确 User-Agent。
- 遵守 robots、服务条款和来源阻断。
- 遇到验证码、登录墙或明确禁止时暂停来源，不绕过限制。

## 测试样本

每个适配器保存：

- 一个正常响应样本。
- 一个空榜单样本。
- 一个字段缺失或结构变化样本。
- 一个超大或异常内容测试。

样本移除 Cookie、Token、用户标识和无关正文。测试只读取样本，不访问实时来源。

## 运行指标

- 最后尝试和最后成功时间。
- 请求耗时。
- 候选数量。
- 去重数量。
- 解析失败数量。
- 连续失败次数。

来源超过 6 小时未成功时告警，但不将 Web 标记为不可用。
