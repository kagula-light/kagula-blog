# Live2D 看板娘

本文定义“神乐静无月”看板娘的 V1 技术、体验、授权和验收边界。它是公开站点的可选增强，不是页面可用性的前置依赖。

## 目标

- 用原创角色建立持续但克制的陪伴感。
- 在不阻塞 LCP、不遮挡阅读、不强迫移动端下载大资源的前提下提供 Live2D 动画。
- 运行时或模型失败时保留同角色静态海报和完整页面能力。
- 让用户可以休眠、关闭并持久保留选择。

V1 不包含 AI 聊天、语音识别、远程指令、付费服装系统或用户上传模型。

## 当前实现状态

运行时边界、服务器配置校验、关闭偏好、欢迎场景完成事件、桌面空闲加载、移动端/减少动画显式启动、4 秒加载超时、页面隐藏休眠和确定性销毁已经实现。`l2d-widget` 固定为 `0.1.0`，只有 `load-mascot.ts` 动态导入该包；生产默认 `MASCOT_ENABLED=false`。

当前仓库没有已授权的原创 Live2D 模型。Playwright 使用故意不存在的 R2 相对路径验收错误回退；`kagura-avatar.webp` 是无角色星空占位。因此当前只能声明运行时边界和静态回退完成，不能声明角色动画、非空 Canvas 或模型视觉验收完成。

## 运行时选型

首选 [`hacxy/l2d-widget`](https://github.com/hacxy/l2d-widget)，包名为 `l2d-widget`。2026-07-10 核对的 npm 元数据版本为 `0.1.0`、MIT、ESM 与 TypeScript 类型，且没有声明运行时或 peer 依赖。其接口提供模型切换、休眠和销毁生命周期，适合封装成独立 Next.js Client Component。

| 方案 | 结论 | 原因 |
| --- | --- | --- |
| `hacxy/l2d-widget` | V1 首选 | MIT、接口聚焦、无框架依赖，具备 `sleep()` 与 `destroy()` 等生命周期能力 |
| `stevenjoezhang/live2d-widget` | 不选 | 生态成熟但为 GPL-3.0，集成方式更偏传统全局脚本 |
| `pixi-live2d-display` | 备用 | MIT、控制力更强，但需 PixiJS 和更多 Cubism 运行时集成工作 |

依赖版本已经由 `pnpm-lock.yaml` 固定。若首选库无法满足 CSP、销毁、性能或模型格式要求，再记录新 ADR 切换到低层方案。

## 授权边界

运行时开源许可证与角色模型权利是两件事：

- `l2d-widget` 的 MIT 许可证只覆盖该库，不授予任何模型、纹理、动作、表情、声音或角色权利。
- “神乐静无月”的立绘、拆分图、Live2D 工程、导出模型、纹理、动作和表情必须原创或取得明确的网络发布及商业使用授权。
- 不使用游戏拆包模型、动漫角色模型、来源不明的模型仓库或仅允许个人展示的素材。
- 资产清单记录作者/生成工具、创建日期、原始来源、许可证或合同、允许用途、修改记录和文件哈希。
- 发布前复核当时有效的 Live2D Cubism SDK 许可与发布条件；官方条款变化以发布时版本为准。

## 组件边界

已实现目录：

~~~text
apps/web/src/features/mascot/
  components/
    mascot-shell.tsx       服务端可渲染的稳定占位与海报
    mascot-client.tsx      唯一的 Live2D Client Component 边界
  hooks/
    use-mascot-runtime.ts  加载状态、事件和卸载清理
  lib/
    load-mascot.ts         动态导入与第三方运行时适配
    mascot-preferences.ts  本地偏好读取与写入
  types.ts                 生命周期和稳定配置类型
~~~

根布局只组合一次 `MascotShell`；第三方 API、浏览器对象和清理逻辑只存在于 `features/mascot`。模型文件不提交进 Git，发布到自有 R2 资源域。第三方组件自行挂到 `document.body` 的宿主会被适配器标记，站内 CSS 只通过该标记约束安全区、尺寸和层级。

## 加载流程

1. Server Component 输出固定尺寸占位、静态海报和可访问的启动按钮。
2. 客户端读取减少动画、设备级策略和已保存的关闭偏好。
3. 桌面端仅在欢迎场景结束、页面可交互且浏览器空闲后动态导入运行时。
4. 移动端或减少动画模式不自动请求模型，只有用户显式启动才加载。
5. 运行时读取自有 R2 上的模型清单、纹理、动作和表情。
6. 成功后从海报过渡到 Canvas；失败则保留海报并停止重试风暴。
7. 路由卸载或功能关闭时调用 `destroy()`，清理监听器、计时器、Canvas 和动画循环。

不得把运行时或模型加入首屏 preload，不得让欢迎场景等待看板娘加载。

## 状态与偏好

生命周期使用有限状态：

- `POSTER`：静态回退可见，尚未请求模型。
- `LOADING`：运行时或模型正在加载，容器尺寸不变。
- `ACTIVE`：模型可见并允许有限交互。
- `SLEEPING`：停止非必要动作，保留重新唤醒入口。
- `DISMISSED`：本地偏好要求不再自动显示。
- `ERROR`：加载失败，回到静态海报并记录非敏感诊断。

关闭偏好保存在浏览器本地；欢迎场景仍按会话存储，两者不能共用一个状态键。站点更新模型版本时不得擅自清除用户关闭偏好。

## 交互与布局

- 桌面端放在右下安全区，使用固定宽高和 `env(safe-area-inset-*)`。
- 休眠、关闭和重新唤醒使用可聚焦按钮、可访问名称和 Tooltip。
- 气泡只显示简短问候或页面状态，不承载关键导航、错误恢复和业务操作。
- 气泡、模型和热区都不得覆盖文章正文、吸附目录、评论输入和 Cookie/权限提示。
- 移动端默认只有小型启动入口；用户启动后仍限制占用范围，并允许立即关闭。
- `prefers-reduced-motion` 下使用静态海报，不自动播放动作、口型、粒子和入场过渡。

## 性能与失败策略

- 首屏 HTML、CSS 和主要内容不依赖看板娘成功。
- 静态海报使用 AVIF/WebP 和明确尺寸，避免布局偏移。
- 模型包按动作和表情拆分，首轮只取待机所需资源；实际预算在原创模型产出后记录并由 CI 检查。
- 运行时加载失败、模型 404、纹理失败、WebGL/Canvas 不可用时都降级为海报。
- 自动加载每次会话最多尝试一次；显式重试由用户触发。
- 页面隐藏或看板娘休眠时停止非必要帧循环和定时器。

## 安全

- 生产运行时由锁文件和构建产物提供，不从公共 CDN 执行远程脚本。
- 模型路径必须是配置的 R2 资源根下相对路径，拒绝任意协议、域名和路径穿越。
- 发布脚本校验资源清单、允许扩展名、文件大小、总大小和哈希。
- CSP 仅增加实际需要的脚本、Worker、WASM 和资源域，不使用通配符解决加载错误。
- 不把第三方响应或模型元数据当作可执行指令，不在日志中记录签名 URL 或 Secret。

## 验收

- 桌面端正常网络下，欢迎场景结束后页面先可交互，看板娘随后加载。
- 移动端和减少动画模式在用户点击前没有模型资源请求。
- 模型 Canvas 非空、裁切正确、可休眠、可关闭，刷新后保留偏好。
- 模型 URL 失败、禁用脚本和 Canvas 不可用时静态海报仍可见，页面无功能损失。
- 360、390、768、1280 和 1440 宽度下不遮挡正文及主要控件。
- 路由切换后没有重复 Canvas、监听器、计时器或持续帧循环。
- 依赖许可证和全部模型资产权利记录齐全。

浏览器验收使用 Playwright 截图、网络请求断言、控制台错误检查和 Canvas 像素检查。真实模型接入前，只能声明组件和回退链路通过，不能声明 Live2D 视觉完成。
